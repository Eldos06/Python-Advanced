## 2. Разбор концепта: Ограниченный `TypeVar`

Основная ценность этого примера — использование `TypeVar` с ограничениями (constraints):

Python

```python
from typing import TypeVar, Any

AddParam = TypeVar('AddParam', float, str, list[Any])
```

### В чем разница между `TypeVar` и `Union` (`float | str | list[Any]`)?

- **`Union` (ИЛИ):** Если мы напишем `def add(a: float | str, b: float | str)`, то Python разрешит передать в `a` строку, а в `b` — число. Но сложить их `a + b` нельзя! Более того, возвращаемый тип тоже будет «смешанным».
    
- **`TypeVar` (Обобщение):** Работает как **переменная для типа**. Она гарантирует **строгое совпадение**. Если на вход `a` пришел тип `str`, то `b` обязан быть `str`, и функция вернет **только** `str`.
    

## 3. Анализ функций и поведения Mypy

### Функция `hello` (Базовая типизация)

Python

```python
def hello(name: str) -> str:
    line = f"Hi, {name}!"
    return line
```

Обычная строго типизированная функция.

🛑 **Раскомментированные ошибки:**

Python

```python
print(hello(2341))      # Ошибка Mypy: Argument 1 to "hello" has incompatible type "int"; expected "str"
print(hello(b'dwin'))   # Ошибка Mypy: Argument 1 to "hello" has incompatible type "bytes"; expected "str"
```

### Функция `add` (Сохранение контекста типа)

Python

```python
def add(a: AddParam, b: AddParam) -> AddParam:
    return a + b
```

Благодаря `AddParam`, Mypy понимает, что:

- `add(1, 2)` -> принимает `float` (в Python `int` является подтипом `float`) и возвращает `float`.
    
- `add('sdf', 'weqr')` -> принимает `str`, возвращает `str`.
    

### Функция `repeat_item` (Динамические списки)

Python

```python
def repeat_item(item: AddParam, n: int) -> list[AddParam]:
    return [item] * n
```

Функция возвращает список элементов того же типа, который был передан на вход.

Давайте посмотрим на этот участок кода:

Python

```python
items = repeat_item('string', 3)  # Mypy выводит тип для items как list[str]
print(items)

items.append('qwerty')  # Без проблем: добавляем строку в список строк
# items.append(12345)   # Ошибка Mypy: Argument 1 to "append" of "list" has incompatible type "int"; expected "str"
```

**Почему это важно:** Mypy «запомнил», что `items` — это строго `list[str]`, потому что в `repeat_item` мы передали строку. Попытка добавить туда число `12345` пресекается анализатором, защищая от `TypeError` в будущем.

## 📌 Главные выводы лекции

1. **`TypeVar` связывает типы** аргументов и возвращаемого значения между собой.
    
2. **Ограничения в `TypeVar`** (`float, str...`) жестко лимитируют список типов, с которыми может работать generic-компонент.
    
3. **Mypy в режиме `strict`** защищает динамические структуры данных (такие как списки) от случайного загрязнения объектами чужих типов в процессе работы программы.
