# Системы контроля зависимостей: Poetry

Файл `pyproject.toml` — это единый стандарт конфигурации для Python-проектов, который используется Poetry (начиная с версии 2.0 используется новый стандарт секции `[project]`).

## 1. Метаданные проекта `[project]`
Секция содержит основную информацию о пакете:
* **name**: `"pythonadvanced"` — имя вашего проекта/пакета.
* **version**: `"0.1.0"` — текущая версия (соблюдает Semantic Versioning).
* **description**: `""` — краткое описание проекта.
* **authors**: Список авторов проекта в формате словарей с именем и email.
* **requires-python**: `">=3.13"` — минимально поддерживаемая версия Python. Проект требует Python 3.13 или выше.

## 2. Основные зависимости `dependencies`
Список библиотек, необходимых для работы приложения в продакшене:

* `aiohttp (>=3.14.1,<4.0.0)` — Асинхронный HTTP клиент/сервер. Ограничение версии: от 3.14.1 включительно до 4.0.0 (не включая).
* `mypy (>=2.1.0,<3.0.0)` — Статический анализатор типов для Python.
* `fastapi[all] (>=0.136.3,<0.137.0)` — Современный быстрый веб-фреймворк. Модификатор `[all]` означает установку со всеми дополнительными зависимостями (включая pydantic, email-validator и др.).
* `uvicorn (>=0.23.0)` — ASGI-сервер для запуска FastAPI. Разрешена любая версия выше или равная 0.23.0.

## 3. Группы зависимостей `[dependency-groups]`
Используется для разделения окружений (например, зависимости только для разработки или тестирования).

* **Группа `dev`**: Зависимости, нужные только разработчику на локальной машине.
    * `pytest (>=9.0.3,<10.0.0)` — Фреймворк для тестирования кода.

## 4. Настройки сторонних инструментов `[tool.mypy]`
Конфигурация для статического анализатора **Mypy**:
* `python_version = "3.13"` — Mypy будет проверять код на соответствие синтаксису Python 3.13.
* `strict = true` — Включает строгий режим проверки (запрещает нетипизированные функции, неявные `Optional` и т.д.). Рекомендуется для продвинутой разработки.

## 5. Сборщик проекта `[build-system]`
Техническая секция, указывающая, как именно собирать проект в `.whl` или `tar.gz`.
* **requires**: `["poetry-core>=2.0.0,<3.0.0"]` — для сборки используется облегченная версия Poetry (версии 2.x).
* **build-backend**: `"poetry.core.masonry.api"` — стандартный бэкенд сборки от Poetry.

## 💡 Шпаргалка по командам Poetry для этого проекта

- **Установка всех зависимостей** (включая dev):
```bash
  poetry install
```



- **Установка только production-зависимостей** (без pytest):
    
    Bash
    
    ```
    poetry install --only main
    ```
    
- **Запуск линтера Mypy** внутри виртуального окружения:
    
    Bash
    
    ```
    poetry run mypy .
    ```
    
- **Запуск тестов** через pytest:
    
    Bash
    
    ```
    poetry run pytest
    ```
    
- **Добавление новой production-зависимости**:
    
    Bash
    
    ```
    poetry add <package_name>
    ```
    
- **Добавление зависимости в dev-группу**:
    
    Bash
    
    ```
    poetry add <package_name> --group dev
    ```
