# Алгоритмы и структуры данных — практика

Набор классических алгоритмических задач (в основном черновики, закомментированы построчно по мере решения) -
сортировки, бинарный поиск по ответу, поиск ближайшего значения.

## 1. Quicksort — [[test data structure/homework2.py|homework2.py]]

```python
def qsort(a):
    if len(a) < 2:
        return a
    p = a[0]
    left = [x for x in a[1:] if x <= p]
    right = [x for x in a[1:] if x > p]
    return qsort(left) + [p] + qsort(right)
```

Классическая рекурсивная сортировка через выбор опорного элемента (`p = a[0]`) и разбиение остальных на "меньше
либо равно" и "больше". Комментарий рядом верно приводит сложность: `O(n log n)` в среднем, `O(n²)` в худшем
случае (если массив уже отсортирован и опорный элемент всегда крайний).

## 2. Бинарный поиск по ответу — "агрессивные коровы"

```python
def canPlace(stall, dist, k):           # можно ли расставить k коров на расстоянии >= dist друг от друга?
    cntCow, last = 1, stall[0]
    for i in range(1, len(stall)):
        if stall[i] - last >= dist:
            cntCow += 1
            last = stall[i]
    return cntCow >= k

def aggressiveCow(stall, k):
    stall.sort()
    low, high, ans = 1, stall[-1] - stall[0], -1
    while low <= high:
        mid = (low + high) // 2
        if canPlace(stall, mid, k):     # если dist=mid достижимо - пробуем увеличить дистанцию
            ans, low = mid, mid + 1
        else:
            high = mid - 1              # иначе уменьшаем
    return ans
```

Классический паттерн "бинарный поиск по ответу": вместо поиска элемента в массиве ищем **значение параметра**
(здесь - максимальную минимальную дистанцию), для которого проверочная функция (`canPlace`) меняет ответ с
True на False. Работает только если проверочная функция монотонна относительно параметра.

Тот же паттерн, но для вещественных чисел (метод бисекции для уравнения):

```python
l, r = 0.0, math.sqrt(C) + 10
for _ in range(100):                     # фиксированное число итераций вместо условия на точность
    mid = (l + r) / 2
    if mid**2 + math.sqrt(mid) < C:
        l = mid
    else:
        r = mid
print(f"{(l + r) / 2:.9f}")              # найти x, при котором x² + √x == C
```

## 3. Merge sort + удаление дубликатов

```python
def mergeSort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left, right = mergeSort(arr[:mid]), mergeSort(arr[mid:])
    res, i, j = [], 0, 0
    while i < len(left) and j < len(right):
        if left[i] < right[j]:
            res.append(left[i]); i += 1
        else:
            res.append(right[j]); j += 1
    res.extend(left[i:]); res.extend(right[j:])
    return res
```

Стандартная реализация "разделяй и властвуй": делим массив пополам, сортируем половины рекурсивно, сливаем два
отсортированных списка за один проход (`merge`). В соседнем черновике этот же `mergesort` комбинируется с
`unique(n)` (убрать дубликаты, сохраняя порядок первого вхождения) для задачи "найти второй по величине уникальный
элемент" (`n[-2]` после сортировки и дедупликации).

## 4. Поиск ближайшего значения — [[test data structure/homework2.py|homework2.py]] (задача 7, активный код)

```python
def closest(myList: list, target: float):
    f = int(target // 1)               # целая часть target
    if f in myList:
        currentIndex = myList.index(f)
        if currentIndex + 1 < len(myList):
            next_val = myList[currentIndex + 1]
            if abs(next_val - target) < abs(f - target):
                return next_val          # сосед справа ближе, чем сама целая часть
        return f
    else:
        return min(myList, key=lambda x: abs(x - target))   # общий случай: линейный перебор ближайшего

print(closest([1, 2, 3, 4, 5], 3.7))  # 4
print(closest([1, 2, 3, 4, 5], 2.2))  # 2
print(closest([1, 2, 3, 4, 5], 4.5))  # 5 (равное расстояние до 4 и 5 - берёт следующий)
```

`min(iterable, key=...)` - идиоматичный способ найти элемент с минимальным значением произвольной функции
(здесь - расстояния до `target`), без ручного цикла со сравнением.
