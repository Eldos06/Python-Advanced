# Управление памятью и Garbage Collection (GC) в Python

## 1. Подсчет ссылок (Reference Counting) — Основной механизм

В CPython (стандартной реализации Python) основным механизмом управления памятью является **подсчет ссылок**. Каждый раз, когда создается ссылка на объект, его счетчик увеличивается. Когда ссылка удаляется — уменьшается. Как только счетчик падает до `0`, память мгновенно освобождается.

Для проверки количества ссылок используется функция `sys.getrefcount(obj)`.

> [!WARNING] Важно помнить
> 
> `sys.getrefcount(obj)` всегда возвращает значение **на 1 больше**, чем есть на самом деле, потому что сама функция `getrefcount` временно создает еще одну (временную) ссылку на объект внутри своего контекста во время выполнения.

### Разбор примеров из кода:

Python

``` python
import sys  
  
name = 'Yeldos'  
print(f"refs of name: {sys.getrefcount(name)}") # Вернет N (больше 1 из-за интернирования строк и аргумента функции)  
  
names = [name, "Dosic"]  
# Счетчик `name` увеличился, так как список теперь тоже ссылается на него.  
print(f"refs of name: {sys.getrefcount(name)}") 
  
del names # Удаляем список -> счетчик name уменьшается обратно
```

## 2. Проблема циклических ссылок

Подсчет ссылок идеален, но он **бессилен против циклических ссылок** (Reference Cycles). Это ситуация, когда объекты ссылаются друг на друга, образуя замкнутый круг. Даже если внешних ссылок на них не останется, их счетчики никогда не упадут до `0`.

### Пример циклической структуры (Связный список / Граф):



``` python
class Node:  
    def __init__(self, value, next_value):  
        self.value = value  
        self.next_value = next_value  
  
node_a = Node('a', None)  
print(f"refs of node_a: {sys.getrefcount(node_a)}")  
node_b = Node('b', node_a) # node_b ссылается на node_a  
print(f"refs of node_a: {sys.getrefcount(node_a)}")  
print(f"refs of node_b: {sys.getrefcount(node_b)}")  
# Если мы сделаем вот так (создадим цикл):  
# node_a.next_value = node_b  
# То node_a ссылается на node_b, а node_b на node_a.  
  
del node_a # Удаляем внешнюю переменную, но объект в памяти жив, пока жив node_b!  
# print(f"refs of node_a: {sys.getrefcount(node_a)}")  
print(f"refs of node_b: {sys.getrefcount(node_b)}")
```

## 3. Модуль `gc` — Сборщик циклического мусора

Для решения проблемы изолированных циклов в Python встроен **Garbage Collector (GC)**, который работает поверх подсчета ссылок. Он ищет группы объектов, которые недостижимы из «внешнего мира», но удерживают друг друга.

### Концепция поколений (Generations)

GC разделяет объекты на **3 поколения** (Generations: 0, 1, 2) на основе их "возраста" (сколько проверок сборщика они пережили):

- **Поколение 0:** Сюда попадают все новые объекты. Проверяется чаще всего.
    
- **Поколение 1:** Сюда перемещаются объекты, выжившие после очистки Поколения 0.
    
- **Поколение 2:** Самое старое поколение. Сюда попадают долгоживущие объекты. Проверяется реже всего.
    

## 4. Главные функции модуля `gc` (Шпаргалка)

Python

```python
import gc
```

- **`gc.collect(generation=None)`**
    
    Вызывает принудительную сборку мусора. Можно передать номер поколения (0, 1 или 2). Возвращает _количество уничтоженных (недостижимых) объектов_.
    
- **`gc.disable()` / `gc.enable()`**
    
    Отключает/включает автоматический сборщик циклического мусора. (Иногда отключают для ускорения критически важных по времени скриптов, где точно нет циклов).
    
- **`gc.isenabled()`**
    
    Возвращает `True`, если авто-сборка включена.
    
- **`gc.get_threshold()`**
    
    Возвращает кортеж из трех пороговых значений `(threshold0, threshold1, threshold2)`. Они определяют, как часто запускается сборка для каждого поколения.
    
- **`gc.get_count()`**
    
    Возвращает текущие счетчики триггеров для каждого поколения `(num0, num1, num2)`. Когда `num0 > threshold0`, запускается сборка поколения 0.
    
- **`gc.get_referents(*objs)`**
    
    Возвращает список объектов, на которые **напрямую ссылаются** переданные объекты. (То, что использовалось в твоем коде).
    
- **`gc.get_referrers(*objs)`**
    
    Возвращает список объектов, которые **ссылаются на** переданные объекты.
    
- **`gc.is_tracked(obj)`**
    
    Проверяет, следит ли GC за объектом. Простые атомарные типы (числа `int`, строки `str`) сборщиком циклов **не отслеживаются**, так как они физически не могут создать цикл. Отслеживаются только контейнеры (списки, словари, кастомные классы).
    

## 5. Область видимости и ссылки (Локальные переменные)

Когда функция завершает работу, её локальная область видимости (стек-фрейм) уничтожается, а значит, уничтожаются и все ссылки на объекты, созданные внутри неё.

Python

```python
def outer_fun(param):
    some_object = object()
    
    def inner_fun():
        return object()
    
    result = inner_fun()
    return result # Передает объект "наружу", сохраняя ему жизнь

new_param = object()
returned_obj = outer_fun(new_param) 
# Память из-под some_object и inner_fun очищена сразу после выхода из outer_fun.
# Жив только returned_obj, так как ссылка на него записана в переменную.
```
