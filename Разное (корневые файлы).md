# Разное — корневые файлы репозитория

Файлы прямо в корне репозитория, не относящиеся к конкретному уроку: конфигурация всего проекта, черновики и
runtime-артефакты (логи, вывод скриптов) от запуска примеров из разных папок.

## Конфигурация всего репозитория

- [[pyproject.toml]] - корневой Poetry-проект `pythonadvanced` (Python `>=3.13`, зависимости `aiohttp`, `mypy`, `fastapi[all]` и др.).
- [[poetry.lock]] - зафиксированные версии зависимостей (генерируется автоматически, не редактируется руками).
- [[requirements.txt]] - альтернативный/экспортированный список зависимостей в формате `pip freeze` (обратите внимание:
  файл сохранён в кодировке с разделителями между каждым символом - типичный артефакт при неверной кодировке экспорта).
- [[dist/pythonadvanced-0.1.0.tar.gz|dist/pythonadvanced-0.1.0.tar.gz]] - собранный Poetry-дистрибутив (`poetry build`) корневого пакета.
- [[.coverage]] - бинарная БД покрытия в корне репозитория, оставшаяся от запуска `coverage`/`pytest --cov` из одной из папок [[10_testing/3_pytest/pytest.md|10_testing]].

## Черновые скрипты в корне

- [[common.py]] - ещё одна копия helper'а `configure_logging` (тот же паттерн, что в [[logging/Логирование.md|logging]] и
  [[1.Вводное занятие/Threading and Multiprocessing.md|1.Вводное занятие]]), лежащая прямо в корне - похоже на случайно
  оставленную рабочую копию.
- [[__init__.py]] - пустой файл в корне репозитория (технически превращает всю папку в пакет, реального смысла не несёт).
- [[intToWords.py]] - отдельный черновик: `detect_days` сопоставляет цифру набору букв казахского алфавита (незакончен, тело `w...` обрывается).
- [[schema.sql]] - пустой файл-заготовка для схемы БД.
- [[test.js]] - несмотря на расширение `.js`, это не код, а экспорт колоды Anki (`#separator:tab`, `#html:true`) с карточкой
  по теме "Asynchronous File I/O" - вероятно, конспект-флэшкарта к [[6.asynchronous/practice/lecture/4/Aiofiles and currency project.md|aiofiles]].

## Runtime-артефакты (вывод запущенных скриптов)

Эти файлы - не код, а результат запуска примеров из других папок; перезаписываются при повторном запуске:

- [[main.log]] / [[myapp.log]] - вывод логов из [[logging/Логирование.md|logging/documantation/main.py]] (`INFO:__main__:Started` → `INFO:mylib:Doing something` → `INFO:mylib:Done someThing` → `INFO:__main__:Finished` - видно по сообщениям, что это именно тот скрипт).
- [[file.log]] - вывод одного из логгеров, пишущих в `FileHandler("file.log")` (формат `%(asctime)s - %(name)s - %(levelname)s - %(message)s`, см. [[6.asynchronous/practice/lecture/2/Async client-server.md|6.asynchronous]] и [[6.asynchronous/practice/lecture/4/Aiofiles and currency project.md|lecture/4]], где встречается такой же формат).
- [[log.txt]] - демонстрация всех пяти уровней логирования подряд (`DEBUG`/`INFO`/`WARNING`/`ERROR`/`CRITICAL`).
- [[email-john@example.com-1754830624.237021.txt]] и [[email-john@example.com-1754830829.9560578.txt]] - письма, сохранённые
  `FileEmailBackend` из [[7.MetaClass/2.Протоколы. Абстрактные классы/ABC & Protocols.md|ABC & Protocols]] (имя файла - `email-{получатель}-{timestamp}.txt`).
