# Подготовка к экзамену — повторение (float/Decimal, str/bytes)

Короткая разминка перед экзаменом, повторяющая темы из [[3.Сложные простые типы/Строки, байты и Decimal.md|3.Сложные простые типы]]:
точность чисел и различие `str`/`bytes`.

## 1. `float` неточен, `Decimal` точен — [[ready_for_exem_1/practice_1.py|practice_1.py]]

```python
0.1 + 0.2 == 0.3                                          # False - двоичное приближение даёт 0.30000000000000004
Decimal("0.1") + Decimal("0.2") == Decimal("0.3")         # True - Decimal хранит точную десятичную дробь
```

> [!WARNING] `Decimal(0.1)` vs `Decimal("0.1")`
> Важно создавать `Decimal` из **строки**, а не из `float`: `Decimal(0.1)` унаследует уже испорченное двоичное
> приближение float'а (`Decimal('0.1000000000000000055511151231257827021181583404541015625')`), а `Decimal("0.1")`
> - ровно `0.1`. В этом файле оба раза используется строковый вариант - правильно.

## 2. `str` + `bytes` нельзя складывать напрямую — [[ready_for_exem_1/practice_2.py|practice_2.py]]

```python
s = "привет"
b = b"hello"
try:
    print(s + b)          # TypeError: can only concatenate str (not "bytes") to str
except:
    print(f'{s} {b.decode()}')   # рабочая версия: сначала decode() у bytes, потом склейка str + str
```

## 3. `cp1251` round-trip — [[ready_for_exem_1/practice_3.py|practice_3.py]]

```python
bytes_cp = "Елдос".encode("cp1251")     # str -> bytes в конкретной кодировке
bytes_cp = bytes_cp.decode("cp1251")    # bytes -> str обратно той же кодировкой
```

Симметрия `encode`/`decode`: чтобы получить исходную строку обратно, нужно расшифровывать той же кодировкой, что
использовалась при кодировании - иначе получится или мусор, или `UnicodeDecodeError` (см. подробнее в
[[3.Сложные простые типы/Строки, байты и Decimal.md|Строки, байты и Decimal]]).
