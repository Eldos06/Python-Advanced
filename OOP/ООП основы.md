# ООП — основы (классы, методы, объекты)

## 1. Класс выполняется при объявлении — [[OOP/practice1.py|practice1.py]]

```python
class name:
  print("Hi Yeldos")

print(type(name))
```

`print("Hi Yeldos")` внутри тела класса выполнится **один раз, сразу при объявлении класса** (на этапе импорта
модуля), а не при создании экземпляра - тело класса это обычный блок кода, который Python исполняет для построения
namespace класса. `type(name)` выведет `<class 'type'>`, потому что сам класс - это тоже объект (экземпляр `type`).

> [!NOTE] Функция вне класса vs метод внутри класса
> Одна и та же `def` внутри класса становится **методом** (первым параметром неявно получает `self`), вне класса -
> обычной функцией. Это просто вопрос того, в чьём namespace объявлена функция.

## 2. Прогрессия одного и того же класса `Point` — [[OOP/practice2.py|practice2.py]]

Файл - последовательность экспериментов (закомментированы один за другим), от самого примитивного к более
правильному ООП:

```python
# Вариант 1: атрибуты добавляются "на лету", без __init__
class Point:
    pass
p1 = Point()
p1.x, p1.y = 5, 4          # никто не мешает добавить атрибут прямо на объекте
```

```python
# Вариант 2: __init__ делегирует работу другому методу (move)
class Point:
    def __init__(self, x, y):
        self.move(x, y)     # конструктор не дублирует логику, а переиспользует метод

    def move(self, x, y):
        self.x, self.y = x, y

    def reset(self):
        self.move(0, 0)      # reset тоже переиспользует move, а не пишет self.x = 0 напрямую
```

```python
# Вариант 3: + вычисление расстояния между точками
def calculate_distance(self, other_point):
    return math.sqrt((self.x - other_point.x) ** 2 + (self.y - other_point.y) ** 2)

assert point2.calculate_distance(point1) == point1.calculate_distance(point2)  # расстояние симметрично
```

Идея прогрессии: чем меньше дублирования (`reset` вызывает `move`, а не копирует его тело) - тем меньше риск,
что при изменении логики `move` придётся не забыть поправить и `reset`.

### Мини-проект: `Note`/`Notebook`

```python
last_id = 0  # общий счетчик id для ВСЕХ заметок, на уровне модуля

class Note:
    def __init__(self, memo, tags=""):
        self.memo, self.tags = memo, tags
        self.creation_date = datetime.date.today()
        global last_id
        last_id += 1
        self.id = last_id   # каждая новая заметка получает следующий id по возрастанию

    def match(self, filter):
        return filter in self.memo or filter in self.tags

class Notebook:
    def __init__(self):
        self.notes = []

    def new_note(self, memo, tags=""):
        self.notes.append(Note(memo, tags))

    def search(self, filter):
        return [note for note in self.notes if note.match(filter)]
```

`Notebook` инкапсулирует список `Note`, а поиск/изменение идёт по `note.id`, а не по индексу в списке - индекс
сдвигается при удалении, id - нет. `global last_id` - простой (не потокобезопасный) способ генерировать
уникальные id без внешней базы данных.

## 3. Незаконченный черновик — [[OOP/lab2_flashCard.py|lab2_flashCard.py]]

```python
class User:
    IdUser: int
    name: str

    def createDesk(self):
```

> [!WARNING] Метод без тела
> `createDesk(self):` не имеет тела (ни `pass`, ни `...`, ни реализации) - в текущем виде этот файл **не
> импортируется**: Python выбросит `IndentationError: expected an indented block` при попытке его загрузить.
> Похоже на незавершённый черновик лабораторной работы.
